## 飞友之家 v3.14b — 修复聊天/消息加载失败（数据库迁移缺失）

**根因**：`/api/chat-list` 查询 `avatar_frame` 列报 500 错误 `column "avatar_frame" does not exist`。
`db.js` 中 `CREATE TABLE IF NOT EXISTS` 不会为已存在的旧表添加新列，且缺少对应的 `ALTER TABLE ADD COLUMN` 迁移语句。

**修复**（db.js）：
- 新增 `avatar_frame` 列迁移：`ALTER TABLE users ADD COLUMN IF NOT EXISTS avatar_frame INTEGER DEFAULT 0`
- 新增 `blocked_users` 列迁移：`ALTER TABLE users ADD COLUMN IF NOT EXISTS blocked_users JSONB DEFAULT '[]'`
- 新增 `banned` 列迁移：`ALTER TABLE users ADD COLUMN IF NOT EXISTS banned BOOLEAN DEFAULT false`
- 新增 `message_type` 列迁移：`ALTER TABLE messages ADD COLUMN IF NOT EXISTS message_type TEXT DEFAULT 'text'`

**验证**：
- `/api/chat-list` → 正常返回好友和群聊列表
- `/api/messages/private/:userId` → 正常返回私聊消息

## 飞友之家 v3.23 — 性能优化：加载速度大幅提升

**问题**：每次打开网站加载很慢，特别是冷启动时。

**根因分析**：
1. Render 免费实例 15 分钟无请求后自动休眠，唤醒需 30-50 秒
2. app.js 174KB 无压缩，每次完整下载
3. 无缓存策略，浏览器每次都重新下载所有资源
4. i18n.js 43KB + style.css 79KB 同样无压缩

**修复**（server.js + package.json）：

| 优化项 | 改动 | 效果 |
|--------|------|------|
| Gzip/Brotli 压缩 | 添加 `compression` 中间件，512B 以上自动压缩 | 174KB app.js → ~40KB (↓77%) |
| 缓存策略 | `maxAge` 1天默认，JS/CSS 添加 `immutable`；图片 7 天；HTML 不缓存 | 二次访问几乎秒开 |
| 防休眠 | 每 10 分钟自 ping `/api/health` | 有人访问后 10 分钟内其他人也秒开 |
| 健康检查 | `GET /api/health` 端点 | 可用于 UptimeRobot 等外部监控 |

> ⚠️ 自唤醒机制仅在实例已唤醒时生效。建议额外使用 **UptimeRobot**（免费）每 5 分钟 ping 一次 `https://liaoliao-chat.onrender.com/api/health`，彻底解决冷启动问题。

**部署**：提交 `509ff35`，已推送 GitHub，Render 自动部署。

## 飞友之家 v3.25 — 优化聊天列表 API 查询速度

**问题**：登录后聊天列表骨架屏持续约 10 秒才显示实际内容。

**根本原因**：`/api/chat-list` 使用 N+1 查询模式——每个好友/群聊分别发起多次数据库查询。好友/群聊数量越多，查询次数线性增长，在 Render 免费实例+Supabase 网络延迟下会达到 10 秒以上。

**修复**：
1. **server.js**：重写 `/api/chat-list`，用 `WITH + LATERAL JOIN` 把所有查询合并为两条 SQL（好友一条、群聊一条），并计算最后一条消息、未读数
2. **db.js**：添加 `messages` 表复合索引
   - `(type, target_id, created_at DESC)` — 加速最后一条消息查询
   - `(type, target_id, sender_id, is_read)` — 加速未读数查询
3. **缓存版本**：`v=328`

**效果**：聊天列表 API 从原来的 10 秒级别降至 1 秒以内（具体取决于数据量和网络）。

**部署**：提交 `23cdaa4`，已推送 GitHub，Render 自动部署。
